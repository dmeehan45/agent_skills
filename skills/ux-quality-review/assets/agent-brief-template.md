## Contract constraints
- Use category names exactly as defined in `plan.md`.
- Use severity labels: Critical, High, Medium, Low.
- Keep finding fields in required order:
  1. ID
  2. Category
  3. Severity
  4. Evidence
  5. User Impact
  6. Recommendation
  7. Implementation Hint

## Steps
1. Inspect assigned category checks.
2. Record observable evidence only.
3. Score category using rubric anchors.
4. Draft findings and prioritized fixes.
5. Add acceptance checks tied to observed issues.

## Done definition
- At least one high-value finding (or explicit “no issue found” with evidence).
- Every finding includes user impact and concrete recommendation.
- Output slots directly into `assets/review-output-template.md`.
